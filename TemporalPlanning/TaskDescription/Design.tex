\subsubsection{Design}

Before embarking on the real implementation, preliminary designs are 
sketched to enable more dynamic development while maintaining a unified 
and clear application architecture.

\textbf{DT1. Uses Case Diagram}

Draw all use cases that the application offers to the user, describing each of them and outlining the relationship between them.

\textbf{DT2. Conceptual Model Diagram}

Design the database content, clarifying all entities,  their attributes and the relationships between them.

\textbf{DT3. Technology Stack Decision}

Select the technology to be used for each application layer.

\textbf{DT4. Logic and Physical Architecture Design}

Design the logical architecture and describe how the implementation code is structured, including the purpose and role of each application layer. This task also includes designing the physical architecture, explaining how the application connects to external services and resources, both physical and virtual, and the function of each of them.

\textbf{DT5. UI mockups Design}

Design the overall style and appearance of the application, which basically corresponds to the UI.

\textbf{DT6. Security Design}

Design a secure user authentication, information submission, storage, and access control.
